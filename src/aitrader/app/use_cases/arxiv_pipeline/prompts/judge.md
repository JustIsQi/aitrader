## 输出要求

只输出**严格的 JSON 对象**（response_format=json_object），字段如下：

```json
{{
  "rating": 1-5,                    // 5 = 强烈推荐做 A 股策略；3 = 可以试试；1 = 完全不适合
  "suitable": true | false,         // rating >= 3 时为 true
  "core_idea": "一句话概括论文核心信号或方法",
  "reasons": ["条目1", "条目2", "条目3"],   // 为什么这个评分
  "data_requirements": [             // 需要的数据，限制在 A 股可得范围
    "日频 OHLCV", "换手率", "估值 PE/PB" 等
  ],
  "ashare_adjustments": [            // A 股映射时需要的改造
    "原文用美股股票池 → 改主板高流动性 120 只",
    "原文是日内 → 改成日频信号，每周一调仓"
  ],
  "risks": ["条目1", "条目2"]        // 风险与限制
}}
```

判分标准（rating）：

- 5 ★：信号清晰、只用日频公开数据、A 股机制不冲突、有强 alpha 或风控价值
- 4 ★：核心可复现，需要少量改造（如换股票池 / 简化频率）
- 3 ★：可做粗研究但需要相当幅度的近似，结果可能与论文不一致
- 2 ★：依赖 A 股不易获取的数据（高频 tick / 期权 IV / 另类数据），或需做空 / 杠杆
- 1 ★：跟 A 股完全不沾边（如美股微观结构、加密货币、宏观经济预测）

绝对不要在 JSON 之外输出任何文本（包括 markdown 代码围栏）。
